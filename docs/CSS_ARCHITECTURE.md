# CSS Architecture

推理ゲームMVPのスタイル方針です。

## 基本方針

- CSS Modulesを使用
- 画面ごとの最小スタイルを優先
- ロジックと同様に、後で差し替えやすいクラス設計にする

## 現在の構成

- `src/app/page.module.css`: ゲーム画面のUIスタイル
- `src/app/globals.css`: グローバルなベーススタイル

## 命名ガイド

- ブロック単位: `container`, `board`, `card`
- 役割ベース: `primaryButton`, `logs`, `success`, `fail`
- 1ファイル内で意味が追える命名にする

## レイアウト方針

- PC: 1カラムで情報を上から順に表示
- モバイル: セレクトUIは `@media` で縦積みに変更
- MVPは可読性優先、装飾は最小限

## アクセシビリティ方針

- ボタン・セレクトは標準要素を利用
- 色だけに依存せず文言（正解/不正解）も表示
- キーボード操作可能なフォーム構成を維持
